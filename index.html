nehan.js

nehan.js is dynamic and logical html layout engine written in javascript, especially focused on generating logical paged media asynchronously.

Supporting not only horizontal layout, but also vertical layout.

Furthermore, flexible functional css value is supported, so you can define styles in very readable and programmable way.

View the Project on GitHub tategakibunko/nehan.js

This demo is generated by jquery.nehan, is easy to use wrapper library of nehan.js.

Install

Include css, js in the <head>.

<link href="/path/to/nehan.css" type="text/css" rel="stylesheet">
<script src="/path/to/nehan.js" type="text/javascript"></script>

Quick Start

Here is super simple example of single page layout using NehanPagedElement class.

var pe = Nehan.createPagedElement();
pe.setStyle("body", {
  "flow":"lr-tb", // or 'tb-rl'
  "width":640,
  "height":480,
  "font-size":16 // camel-case fontSize is not allowed!
});

pe.setContent("<h1>hello, nehan.js</h1>");
document.getElementById("result").appendChild(pe.getElement());
document.getElementById("next-button").onclick = function(){ pe.setNextPage(); };
document.getElementById("prev-button").onclick = function(){ pe.setPrevPage(); };

Getting Started

Here is more precise example using engine interfaces.

  1. Generate layout engine.

    var engine = Nehan.setup();

  2. Setting styles

    Your own styles can be defined by using setStyle(or setStyles).

    See styling for more details.

    engine.setStyles({
  // define root page size(=body).
  // in nehan.js, you can define each properties as functional value.
  "body":{
    "flow":"lr-tb", // normal horizontal mode
    //"flow":"tb-rl", // japanese vertical mode
    //"flow":"tb-lr", // chinese vertical mode
    //"width":800, // direct size
    //"width":"80%", // percentage size from parent(screen.width at this case)
    "width":function(context){
      return Math.floor(screen.width * 70 / 100); // you can return size by funciton.
    },
    "height":"60%",
    "font-family":"'Hiragino Mincho Pro','HiraMinProN-W3','IPA Mincho','Meiryo','MS Mincho',monospace",
    "font-size":function(context){
      ver height = Math.floor(screen.width * 60 / 100);
      return Math.max(12, Math.floor(height / 25));
    }
  },
  // always break the page before h1.
  "h1":{
    "break-before":"always",
    "font-size":"32px"
  },
  // all header types are bolded.
  // you can write element name as regular expression like this.
  "/h[1-6]/":{
    "font-weight":"bold"
  },
  // if external link, highlight background-color.
  "a[href^=#]":{
    "background-color":"gold"
  },
  // make first-letter larger and floated to start.
  // notice that in nehan.js, all class names must be started with 'nehan-'.
  ".nehan-drop-caps::first-letter":{
    "display":"inline-block",
    "box-sizing":"content-box",
    "float":"start",
    "font-size":"4em",
    "measure":"1em",
    "extent":"1em"
  },
  // functional value
  // context:SelectorPropContext
  ".nehan-stripe-list li":{
    "background-color":function(context){
      return (context.getChildIndex() % 2 === 0)? "pink" : "white";
    }
  },
  // special callback argument 'onload', called after loading all css about this selector.
  // context:SelectorContext
  ".nehan-must-be-square img":{
    "onload":function(context){
      var width = context.getCssAttr("width", 100);
      var height = context.getCssAttr("height", 100);
      if(width !== height){
	var larger = Math.max(width, height);
	return {width:larger, height:larger};
      }
    }
  },
  // special callback argument 'oncreate', called after layout tree is evaluated,
  // and dom element is actually created.
  "a.nehan-test-onclick":{
    "background-color":"gold",
    "oncreate":function(dom, context){
      // console.log("you can get StyleContext:%o", context);
      dom.onclick = function(){
        alert("click!");
      };
    }
  }
});

  3. Create page stream

    var stream = engine.createPageStream("hello, nehan.js!");

  4. Yielding page

    Stream is created, now you can yield paged-media by asyncGet.

    stream.asyncGet({
  // called when each single pages are successfully calculated.
  onProgress : function(stream, tree){
    var page = stream.getPage(tree.pageNo); // get page object of this page.
    console.log(page.element);
  },
  onComplete : function(stream, time){
    console.log("finished! %f msec", time);
    // you can create outline element via engine.documentContext.
    var toc = engine.createOutlineElement({
      onClickLink : function(toc){
        console.log("click toc:%o", toc);
      }
    });
  }
});

Styling

As you see at previous chapter(Getting Started), your own style can be defined by setStyle(or setStyles).

But notice that there are some local rules in nehan.js.

Important notices about styling in nehan.js

  1. camel case property is not allowed

    {font-size:"16px"} // OK
{fontSize:"16px"}  // NG!

  2. some properties are defined by 'logical' value

    Following properties are defined by 'logical' value.

    propertyavailable values
    float start, end
    margin start, end, before, after
    padding start, end, before, after
    border-width start, end, before, after
    measure normal size value like px, em, % format.
    extent normal size value like px, em, % format.

    See following example.

    Nehan.setStyle("div.some-class", {
  margin:{before:"10px"},
  float:"start",
  measure:"100px",
  extent:"50px"
});

    In this example,

    If document is horizontal mode(flow = 'lr-tb'), margin.before is 'top', and float.start is 'left', measure is 'width', extent is 'height'.

    If document is vertical mode(flow = 'tb-rl'), margin.before is 'right', and float.start is 'top', measure is 'height', extent measn 'width'.

  3. all id name and class names are added "nehan-" prefix.

    to avoid css collision that is not layouted by nehan.js, all id and class names are automatically prefixed by "nehan-".

  4. about functional css value

    Unlike static css file, you can use functional css value for each css property.

    1. functional css vaue

      Callback argument 'context' in functional css value is 'SelectorPropContext'

      It can access some pseudo like value like nth-child, nth-of-type etc.

      // change backgroiund-color by child index.
".nehan-stripe-list li":{
  "background-color":function(property_context){
    return (property_context.getChildIndex() % 2 === 0)? "pink" : "white";
  }
}
    2. special callback functional value onload

      This callback is called after all css properties of this selector are loaded.

      Type of callback argument is SelectorContext.

      This context is 'extended class' of 'SelectorContext', with some extra interfaces that can touch css object,
      because 'onload' is called after all css of matched elements are loaded.

      // force image metrics to square sizing if width and height is not same.
".nehan-must-be-square img":{
  "onload":function(selector_context){
    var width = selector_context.getCssAttr("width", 100);
    var height = selector_context.getCssAttr("height", 100);
    if(width !== height){
      var larger = Math.max(width, height);
      return {width:larger, height:larger};
    }
  }
}
    3. special callback value oncreate

      Special callback oncreate is called after layout tree is evaluated and dom element is created.

      Argument of this callback is dom element that is created for this selector.

      "a.nehan-test-onclick":{
  "oncreate":function(dom, style_context){
    dom.onclick = function(){ alert("click!"); };
  }
}

  5. special properties in nehan.js

    1. line-rate:(float)

      In normal html, size of 'line-height:1.0em' is determined by font size of 'parent' block.

      In contrast, property line-rate is determined by max font size of 'current line'.

      Assume that font-size of parent block is 16px, and max font size of current line is 32px, line-height:1.0em is 16px, but line-rate:1.0em is 32px.

    2. box-sizing:[content-box | border-box | margin-box(default)]

      In box-sizing, margin-box is special value in nehan.js, and is box-sizing default value.

      In margin-box, even if margin is included in box-size.

      Why? In normal html, outer size of box can be expanded, but in paged layout, outer size is strictly fixed.

      So if you represent margin/border/padding(called in edge in nehan.js), the only way is 'eliminating content space'.

    3. flow:[lr-tb | tb-rl | tb-lr | flip]

      This property represent document-mode in nehan.js.

      lr-tb means inline flows 'left to right', block flows 'top to bottom'.

      tb-rl means inline flows 'top to bottom', block flows 'right to left', and so on.

      flip means toggle Layout["hori"] and Layout["vert"].

      for example, assume that Layout["hori"] is "lr-tb", and Layout["vert"] is "tb-rl",

      and current document direction(Layout.direction) is "hori", flow:"flip" means "tb-rl".
